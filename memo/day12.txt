프로그램
	실행되지 않은 상태.

프로세스
	실행된 프로그램.
	자원(메모리, cpu)과 쓰레드로 구성됨 


쓰레드(서버를 이해하기 위해서 배움)
	프로세스 처리 경로.
	프롤세스 내에서 실제 작업을 수행
	종류: 단일쓰레드, 멀티쓰레드
	여태까지 우리는 단일쓰레드였음.(오류가 나면 그 다음줄 진행이 되지 않았으므로)


프로세스: 쓰레드 = 공장: 일꾼 


	-단일 쓰레드 / 비효율적, 안정적이다 
	ex. 매표소/1열 종대/한 사람이 끝날 때까지 기다려야함.
	
	처리 경로를 한 개만 가지고 있기 때문에 직렬적이다.
	동시에 많은 양을 처리하기 힘들기 때문에 상대적으로 비효율적이다.
	하지만 하나의 작업에 문제가 발생하더라도 다른 작업에는 영향을 끼치지 않는다.
	따라서 안정성이 보장되고 설계 시 멀티쓰레드에 비해 쉽다.

	-멀티 쓰레드
	효율적, 처리비용 감소, 설계가 어렵다
	쓰레드 하나의 문제 발생 시 모든 쓰레드 문제 발생
	ex. 음식점/간단한 걸 시켰을 때 식사 메뉴를 먼저 주문 받았더라도 간단한 것이 빠르게 나온다. 
	ex. 결제는 됐는데 주문이 안 됨. 

	하나의 프로세스를 동시에 처리하는 것처럼 보이지만 사실은 매우 짧은 단위로 분할해서 차례로 처리한다.
	여러 개의 처리 경로를 가질 수 있도록 하며, 동시작업이 가능해진다.
	설계하기 굉장히 어려우며, 하나의 쓰레드 문제 발생 시 모든 쓰레드에 문제가 발생하게 된다.
	JAVA 웹 서버가 대표적인 멀티 쓰레드이다. 멀티 쓰레드로 설계했다면, 처리량 증가, 효율성 증가,
	처리비용 감소의 장점이 있기 때문에 단점을 감수하고 설계하는 편이다.

	속도에 대한 질문? 이론상은 똑같으나 좋은 환경을 가진 쪽이 더 빠르다. 


멀티 쓰레드 구현 방법
	핵심 : run() 메소드 재정의

	1. Thread 클래스 상속
	2. Runnable 인터페이스 지정

Runnable r = new Thread2();
Thread t2 = new Thread(r); //Thread(Runnable r)
t2.start();


run -> 자원 / Thread가 수행할 작업을 작성하면 됨
